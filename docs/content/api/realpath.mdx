# realpath

Resolve the canonical absolute path for a path, resolving `.`, `..`, and symlinks.

## Basic usage

```ts
import { realpath } from '@rush-fs/core'

const abs = await realpath('./some/relative/path')
console.log(abs) // e.g. /home/user/proj/some/relative/path
```

## Methods

### `realpath(path)`

**Async.** Returns `Promise<string>` (canonical absolute path).

| Argument | Type   | Description |
| -------- | ------ | ----------- |
| `path`   | string | Path to resolve. |

### `realpathSync(path)`

**Sync.** Returns `string` or throws.

## Performance

Single syscall (with resolution); on par with Node.js. See [Benchmarks](/benchmarks).

## Notes

- **Symlinks**: Fully resolves symlinks in the path. Fails if the target does not exist (same as Node.js).
- **Known gap**: Missing-path errors currently have Node-like messages, but do not expose Node-style `code`, `path`, and `syscall` fields yet.
- **Unsupported Node inputs/options**: `Buffer` paths, `URL` paths, the `encoding` option, `realpath.native`, and the legacy cache option are not part of the current Rush-FS `realpath` surface.
