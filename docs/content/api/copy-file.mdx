# copyFile

Copy a file from `src` to `dest`. Does not copy directory trees.

## Basic usage

```ts
import { copyFile } from '@rush-fs/core'
import { constants } from 'node:fs'

await copyFile('./a.txt', './b.txt')
await copyFile('./a.txt', './b.txt', constants.COPYFILE_EXCL) // fail if dest exists
```

## Methods

### `copyFile(src, dest, mode?)`

**Async.** Returns `Promise<void>`.

| Argument | Type   | Description |
| -------- | ------ | ----------- |
| `src`    | string | Source file path. |
| `dest`   | string | Destination file path. |
| `mode`   | number | Optional. e.g. `COPYFILE_EXCL` to fail if dest exists. |

### `copyFileSync(src, dest, mode?)`

**Sync.** Same arguments; throws on error.

## Notes

- **Directories**: Use `cp` with `recursive: true` for directory trees. `copyFile` only copies a single file.
- **COPYFILE_EXCL**: Use `constants.COPYFILE_EXCL` from `node:fs` (or the numeric value) to avoid overwriting an existing destination.
- **Known gap**: Missing-path and existing-destination errors currently have Node-like messages, but do not expose Node-style `code`, `path`, `dest`, and `syscall` fields yet.
- **Unsupported Node inputs**: `Buffer` paths and `URL` paths are not part of the current Rush-FS `copyFile` surface.
- **Unsupported reflink guarantee**: `COPYFILE_FICLONE_FORCE` currently falls back to ordinary copy instead of requiring copy-on-write support.
